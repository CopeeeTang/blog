## 今天做了什么

早上起来第一件事是继续推进个人博客项目。这个博客的目标很简单：一个能记录每日复盘、技术洞见和论文笔记的地方，轻量、克制、够用就好。

技术选型上选了 Astro，原因很直接——它对内容优先的网站设计得非常好，构建出来是纯静态 HTML，加载快，部署简单。配合 Tailwind 做样式，完全不需要引入多余的运行时依赖。

## Astro 的一些发现

今天深入看了 Astro 的 Content Collections API，感觉设计很优雅。用 Zod 定义 schema，类型推导直接到模板层，写起来比想象中舒服。

```ts
// content.config.ts 里定义 schema
const daily = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    date: z.coerce.date(),
    summary: z.string().optional(),
    tags: z.array(z.string()).default([]),
    lang: z.enum(['zh', 'en']).default('zh'),
  }),
});
```

`z.coerce.date()` 这个细节很贴心——frontmatter 里写 `2026-03-12` 字符串，会自动转成 Date 对象，不用手动处理。

## 用 Claude Code 建站的体感

最大的感受是：**Claude Code 很擅长"执行有明确规格的任务"**。比如「按这个 schema 建四个 content collection 的 sample 文件」，它能直接干完，不需要反复确认。

但对于「这个博客应该长什么样」这类开放性问题，还是需要自己有清晰的想法再交给它执行。工具放大你的执行力，但不能替你做判断。

今天用了 Agent Teams 功能，让多个 agent 并行处理不同任务（SVG 插图、markdown 内容、组件开发），体感上确实快很多，后面会单独写一篇记录这个工作流。

## 小感悟

做一个「只给自己看」的博客，和做一个「给所有人看」的博客，思维方式很不一样。前者更诚实，可以记录困惑和未完成的想法；后者总忍不住要整理成「有结论的文章」。

我想要的是前者。这里的复盘不需要每篇都有漂亮的收尾，记录本身就是目的。
