## 科研

- [ ] 阶段构造数据的方式

### 方案 A：两阶段构造（推荐）

**第一步**：让 o3 做"推理规划"（不带 tools，走 Responses API 拿 reasoning）

```
Prompt: "你是一个流式视频助手，当前场景是[X]，记忆摘要是[Y]。
         你有 observe()/recall()/respond() 三个工具。
         请详细推理：你会做什么决策？为什么？"
→ 拿到显式 reasoning（o3 在非 tool 场景下稳定）
```

**第二步**：让 o3 执行实际 tool call（走 chat.completions）→ 拿到 tool trajectory

**第三步**：合并 → reasoning + trajectory = 完整训练样本

> 好处：绕过了"tool 回合拿不到 reasoning"的限制

### 流式 vs 非流式

实际影响分析：
- **影响较小**：backward / realtime 任务（单问单答）
- **影响较大**：forward 任务（REC/SSR/CRR），多个子问题依赖时间线推进
